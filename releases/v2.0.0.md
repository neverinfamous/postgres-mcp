# v2.0.0 - Introspection, Security Hardening & Code Mode Evolution

**21 new tools, sweeping security hardening, and a fully rebuilt developer experience.** This release marks the most comprehensive expansion and hardening cycle in the project's history.

## Highlights

### 🔍 Introspection & Migration Tracking (12 New Tools)

A complete agent-optimized schema analysis suite with migration audit trail:

- **Schema analysis** — `pg_dependency_graph`, `pg_topological_sort`, `pg_cascade_simulator`, `pg_schema_snapshot`, `pg_constraint_analysis`, `pg_migration_risks`
- **Migration tracking** — `pg_migration_init`, `pg_migration_record`, `pg_migration_apply`, `pg_migration_rollback`, `pg_migration_history`, `pg_migration_status`
- `pg_schema_snapshot` compact mode and extension schema exclusion for reduced payload sizes
- All tools return structured JSON from read-only `pg_catalog` queries

### 🛡️ Security Hardening (26 Changes)

Defense-in-depth applied comprehensively across the entire codebase:

- **SQL injection prevention** — WHERE clause sanitization across 31+ sites, parameterized `information_schema` queries, catalog query parameterization in backup/PostGIS tools
- **Per-tool OAuth scope enforcement** — Tool invocations check token scopes against tool group requirements via `AsyncLocalStorage`
- **Code Mode sandbox hardening** — `Proxy`, `Reflect`, and `Symbol` constructors blocked; static pattern detection expanded
- **HTTP transport** — Streaming body size enforcement, deterministic rate limit cleanup, `Permissions-Policy` header, deprecated `X-XSS-Protection` removed
- **SSL secure default** — `ssl: true` now defaults to `rejectUnauthorized: true` (certificate verification enabled)
- **Admin/PostGIS/Stats SQL hardening** — `sanitizeIdentifier()`, `sanitizeTableName()`, and parameterized queries applied throughout

### ⚡ Performance & Developer Experience

- **Code Mode sandbox optimizations** — `vm.Script` LRU cache (50 entries), `process.memoryUsage.rss()` single-counter measurement, pre-built async IIFE wrapper
- **Performance benchmark suite** — 59 micro-benchmarks across 7 audit domains
- **ServerInstructions build system** — Markdown source + TypeScript build script, eliminating agent editing errors in 480-line template literals
- **Tool filtering shortcuts restructured** — 15 → 16 shortcuts; oversized shortcuts split for IDE compatibility

### 🔧 Reliability: Structured Errors Everywhere

Building on v1.3.0's deterministic error handling foundation:

- **P154 errors now return parseable JSON** — `structuredContent` instead of `isError: true`, enabling AntiGravity to parse structured responses
- **40+ additional error paths** fixed across introspection, migration, cron, citext, ltree, partman, and vector tools
- **Split Schema pattern** applied to 22+ remaining tools for correct MCP JSON Schema generation

### 📦 Dependencies

- `@modelcontextprotocol/sdk` 1.26.0 → 1.27.1
- ESLint 10.0.1 → 10.0.2, `typescript-eslint` 8.56.0 → 8.56.1
- `pg` 8.18.0 → 8.19.0, `@types/pg` 8.16.0 → 8.18.0
- Security audit fixes: `hono`, `rollup`, `minimatch` transitive dependency upgrades
- ToolFilter unit tests: 58 tests mirroring mysql-mcp coverage

### 📖 Documentation

- README/DOCKER_README — Introspection & Migration Tracking promotion, Code Mode emphasis, stale tool counts fixed (206 → 227)
- `.gitignore`/`.dockerignore` audit — Expanded exclusions for build dirs, env configs, test outputs, debug logs

## Removed

- **Worker isolation mode** — Non-functional `worker_threads` sandbox mode deleted (`worker-sandbox.ts`, `worker-script.ts`, `sandbox-factory.ts`)
- **`pg_dependency_graph.includeIndexes`** — Never-implemented parameter removed
- **`base-core` shortcut** — Identical to `starter`, removed

## Breaking Changes

None. All changes are backwards-compatible. Structured error responses provide strictly more information than previous raw exceptions.

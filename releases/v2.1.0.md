# v2.1.0 - Exhaustive Stabilization, Modular Architecture & Performance Hardening

**100% structured error coverage across all 227 tools, modular file splits, SSE backward compatibility, and defense-in-depth performance optimizations.** This release completes the stabilization sweep started in v2.0.0, achieving zero raw MCP error leaks across the entire tool surface.

## Highlights

- **Zero raw MCP error leaks** — Every tool now returns structured `{success, error}` responses for all error paths including empty params, wrong-type numerics, invalid enums, and nonexistent schemas. Over 80 additional Zod validation leaks, NaN coercion failures, and missing try/catch wrappers fixed across all 21 tool groups
- **SSE backward compatibility (#65)** — Legacy SSE clients can now connect via `GET /sse` + `POST /messages`. Dual-transport support: `/sse` uses `SSEServerTransport`, `/mcp` uses `StreamableHTTPServerTransport`
- **Modular file splits** — `schema.ts` (1083 lines) split into `schema/objects.ts` + `schema/views.ts`; `monitoring.ts` (941 lines) split into `monitoring/basic.ts` + `monitoring/analysis.ts`
- **Version SSoT** — Single `VERSION` constant reads from `package.json` at runtime, replacing 3 hardcoded version strings
- **Performance optimizations** — `writeToStderr` O(1) taint-breaking, combined regex fast-path for WHERE validation, HTTP body-size consolidation, Code Mode `createSandboxBindings` reduced by ~600 lines

### Security

- Hardened TruffleHog CI workflow (pinned version, removed `--only-verified`, hard gate)
- `Referrer-Policy: no-referrer` header added
- NPM audit remediation (`@hono/node-server`, `hono`)
- CVE-2026-27904/27903 minimatch ReDoS fix (override ≥10.2.3 + Dockerfile patch)
- Hardened `npm audit` CI gate (removed `continue-on-error`)
- Introspection tool group scope corrected from `WRITE` to `READ`
- Removed local MCP Registry tokens

### Fixed

- 80+ Zod validation leaks across pgcrypto, kcache, partman, cron, citext, ltree, vector, text, performance, schema, JSONB, core, monitoring, partitioning, and transaction tool groups
- Numeric parameter type safety cleanup (81 lint errors resolved — `z.any()` → `z.coerce.number()`)
- `pg_vector_create_index` duplicate index detection now semantic (matches by table+column, not just name)
- `pg_list_partitions` / `pg_partition_info` aligned to P154 error pattern
- `pg_transaction_begin`/`pg_transaction_execute` invalid `isolationLevel` now returns structured errors
- `pg_jsonb_validate_path` output schema fixed for error paths
- Core tool outer catch blocks (14 sites) now route through `formatPostgresError()`
- Stale comments, file headers, and protocol versions cleaned up across codebase

### Dependencies

- `@types/node` 25.3.3 → 25.3.5
- `jose` 6.1.3 → 6.2.0
- Dockerfile: npm-bundled `tar` patch 7.5.8 → 7.5.10

### Removed

- `security-update.yml` Trivy workflow (redundant with Docker Scout in `docker-publish.yml`)
